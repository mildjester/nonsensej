Trying Out Vite

posted: 2023-02-11

Vite

Vite is a tool for compiling JavaScript and CSS, similar to Webpack.
It was created by Evan You, the developer of Vue.js, and seems easy to get started with, just like Vue.js.
https://vitejs.dev/

I had an old site that could no longer compile, so I decided to switch to Vite.
The site itself is simple—it consists of only a single page, and the main requirement is compiling JavaScript and Sass.

Creating a Vite Project

First, create a new Vite project using the following command:

$ npm create vite@latest

It will ask for confirmation, so enter “y”.

Ok to proceed? (y) y

Next, enter the project name.
For this example, I’ll use “hogehoge-prj”.

✔ Project name: … hogehoge-prj

Then, choose the framework.
Since my site doesn’t use a framework, I’ll select “Vanilla”.
(Use the up/down arrow keys to select.)

? Select a framework: › - Use arrow-keys. Return to submit.
❯   Vanilla
    Vue
    React
    Preact
    Lit
    Svelte
    Others

Next, choose between JavaScript and TypeScript.
I’ll go with JavaScript.
(Use the up/down arrow keys to select.)

? Select a variant: › - Use arrow-keys. Return to submit.
❯   JavaScript
    TypeScript

This completes the project setup, and you’ll see the following message:

Scaffolding project in /Users/goro/workspace/hogehoge-prj...

Done. Now run:

  cd hogehoge-prj
  npm install
  npm run dev

If you run these commands, the default page will launch:

Installing Sass

Vite does not support Sass by default, so install it separately:

$ npm add -D sass

Integrating an Existing Site with Vite

Now, I’ll migrate an existing site into the Vite project.

Editing index.html

First, modify index.html to include the page template (HTML structure).
Instead of manually linking compiled JavaScript and CSS, just include main.js at the bottom of the <body>.

Any CSS files placed in the public directory (explained later) should be explicitly linked.

Here is the default index.html:

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <link rel="icon" type="image/svg+xml" href="/vite.svg" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Vite App</title>
  </head>
  <body>
    <div id="app"></div>
    <script type="module" src="/main.js"></script>
  </body>
</html>

Editing main.js

In main.js, import the necessary JavaScript files, Sass files, and images.

By default, the <div id="app"></div> element in index.html is populated with content via JavaScript.However, for static content, it’s better to write it directly in index.html.

For existing sites, main.js only needs to contain import statements for the necessary files.

Here is the default main.js:

import './style.css'
import javascriptLogo from './javascript.svg'
import { setupCounter } from './counter.js'

document.querySelector('#app').innerHTML = `
  <div>
    <a href="https://vitejs.dev" target="_blank">
      <img src="/vite.svg" class="logo" alt="Vite logo" />
    </a>
    <a href="https://developer.mozilla.org/en-US/docs/Web/JavaScript" target="_blank">
      <img src="${javascriptLogo}" class="logo vanilla" alt="JavaScript logo" />
    </a>
    <h1>Hello Vite!</h1>
    <div class="card">
      <button id="counter" type="button"></button>
    </div>
    <p class="read-the-docs">
      Click on the Vite logo to learn more
    </p>
  </div>
`

setupCounter(document.querySelector('#counter'))

Any imported files will have cache busting applied, meaning their filenames will include a random string to prevent caching.For example, the line src="${javascriptLogo}" ensures that the correct (cache-busted) image file is loaded.

Placing Static Files

The public directory holds files that should be placed in the document root as they are. This includes JavaScript or CSS files that don’t need compiling, as well as images that shouldn’t have cache busting applied.

Deleting Unnecessary Files

Since they are not needed, delete the following files:

  • counter.js
  • javascript.svg
  • public/vite.svg

This completes the setup for using Vite!

You can run the local development server with:npm run dev
And compile the project for production with:npm run build
The compiled files will be output to the dist directory.
